Type Shorthands

The package defines and exports shorthands for commonly used types. They are defined as

I64 = Int64
F64 = Float64
CF64 = ComplexF64
VecF64 = Vector{Float64}
VecCF64 = Vector{ComplexF64}
MatF64 = Matrix{Float64}
MatCF64 = Matrix{ComplexF64}
Warning

This is bad practice, but often more convenient when quickly prototyping...

cSpline

Interpolation of complex-valued data using Dierckx.jl. We provide both a functor, and a classical interface.

PConv.cSpline — Type
cSpline(x::AbstractVector{T}, y::AbstractVector; w=ones(length(x)), k=3, bc="nearest", s=0.0)

Struct and Functor for interpolating complex data, using Dierckx.jl's Spine1D. Spline can be evaluated by calling the struct as a function. Regular derivative and integrate functions are avaivible.

Arguments

  • k degree of spline k (1 = linear, 2 = quadratic, 3 = cubic, up to 5)
  • bc{"nearest" (default), "zero", "extrapolate", "error"} Behavior when evaluating

the spline outside the support domain (minimum(x), maximum(x)).

PConv.cspline — Method
cspline(xi, x::AbstractArray, y::AbstractArray; kwargs...)

Complex Interpolation function, by calleing Spline1D for Re and Im seperately. Returns Interpolation of x,y evaluated at xi, if y is a complex array.

Arguments

  • xi::AbstractArray: New points
  • x::AbstractArray{Real}: x-values of the data (ordered and real)
  • y::AbstractArray: y-values of the data ∈ Complex
  • kwargs::NamedTuple: Stanard options for Spline1D
  • return: Spline1D(x, y, bc)(xi)

PConv.generate_sym — Method
generate_sym(Xmin, Xmax, δx)
generate_sym(Xmax, δx)

Generate symmetric array around 0, i.e. X = Xmin, ..., δx, 0, δx, ..., Xmax. If Xmin is not provided, Xmin = -Xmax is used. Note that X[1] != Xmin , X[end] != Xmax is possible if δx does not fit into the interval.

Arguments

  • Xmin(=-Xmax): Minimal value
  • Xmax: Maximal value
  • δx: Spacing
  • return: X, N, N0 (X[N0] = 0.0)

PConv.versionized_path — Method
versionized_path(filename; abspath=PROJECT_ROOT, overwrite = false)

  • abspath::AbstractString=PROJECT_ROOT: Absolute path to use. Default is the

directory of the active project.

  • overwrite::Bool If false, don't versionize the path and potentially overwrite existing

file

  • return: versionized_path::AbstractString
